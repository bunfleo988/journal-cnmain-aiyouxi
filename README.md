# journal-cnmain-aiyouxi

## 项目简介

本仓库用于归档和发布多个独立 HTML 页面。这些页面以静态文件形式存储，可直接通过 GitHub Pages 或其他静态托管服务访问。

仓库本身不针对特定域名、网站或应用，主要用作页面集合的版本管理与分发。

## 目录结构

```
.
├── index.html          # 首页 / 导航页（可选）
├── pages/              # 存放独立 HTML 页面的目录
│   ├── page1.html
│   ├── page2.html
│   └── ...
├── assets/             # 公共资源（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `pages/` 目录下存放各独立页面文件，命名建议保持语义化和唯一性。
- `assets/` 目录用于存放页面间共享的静态资源，避免重复引用。

## 页面归档说明

- 每个 HTML 页面均为自包含或依赖 `assets/` 下公共资源。
- 页面功能、样式、交互逻辑均由各自文件独立定义。
- 若页面需要引用外部资源（如字体库、CDN 库），请在页面内直接声明。
- 不建议在页面间硬编码内部链接，除非有明确的导航需求。

## 维护说明

- 新增页面：在 `pages/` 下创建新的 HTML 文件，并确保引用路径正确。
- 更新页面：直接修改对应 HTML 文件，提交后自动更新。
- 删除页面：移除文件并检查是否有其他页面引用该内容。
- 资源管理：公共资源更新时需注意向后兼容，避免影响已有页面。

## 使用方式

本仓库主要面向归档与展示场景，不提供具体功能或服务。推荐通过以下方式使用：

1. 克隆仓库到本地，在浏览器中直接打开 `index.html` 或 `pages/` 下的文件。
2. 启用 GitHub Pages（Settings → Pages → Source），选择主分支或 `docs/` 目录，即可获得在线访问地址。
